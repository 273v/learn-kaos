The capstone. You've built tools, documents, typed programs, and an agent. Now
`kaos-ui` assembles them into a **complete application**: a FastAPI backend on
`kaos-agents` (SSE streaming, bearer auth, a virtual filesystem per session) and a
Vite + React frontend — with Makefile, Docker, Caddy, and pre-commit included.

The scaffold step runs **offline** here via dry-run, so you can see exactly what you'd
get without writing anything.

```bash
uv run examples/scaffold-app.py
```

## Actually create and run it

Drop `dry_run=True`, or use the CLI:

```bash
kaos-ui new web:spa my-legal-app
cd my-legal-app
make install      # uv (backend) + pnpm (frontend), with supply-chain hardening
make dev          # FastAPI + Vite dev servers
```

:::caution[Running needs a key (and a note on availability)]
The scaffold step is offline and deterministic. **Running** the generated app calls a
real LLM, so it needs a provider API key — and, because the app depends on the full
agent stack, you may need the packages installed from source until everything is on
PyPI. The generated `README.md` and `Makefile` spell out the exact steps.
:::

## The six templates

`kaos-ui` scaffolds more than web apps — see them all in the
[package reference](/reference/packages) and the
[how-to on adding your own template kind](/how-to/test-your-own-snippets):

| Kind | What you get |
|---|---|
| `web:spa` | FastAPI + Vite/React full-stack app (this capstone) |
| `web:api` | FastAPI backend only |
| `dashboard:streamlit` | A Streamlit data dashboard |
| `tui:textual` | A terminal UI app |
| `module` | A new KAOS module package |
| `workflow` | A single-file script |
